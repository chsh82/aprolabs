### 남은 101건의 성격 — 코드 수정으로 해결 어려운 이유

1. **텍스트불일치 34건**
   - 전각공백/공백 혼용 (~10건): segmenter 출력 문제
   - 지문 헤더 content 포함 (~3건): segmenter 문제
   - 문항 번호 혼입/오인식 (~10건): Gemini 추출 품질
   - 5번 선택지 집중 (7건): PDF 구조적 문제 (다음 지문 연결)
   - OCR 오인식 (~4건): Gemini 한계

2. **bracket텍스트불일치 24건**
   - Gemini 재처리 시 inner_text 변동으로 기존 마커와 불일치
   - _try_insert() 앵커 탐색 실패

3. **지문못찾음 21건**
   - Gemini 비결정성: 재처리마다 결과 달라짐
   - 일부는 실제 PDF 구조 문제 (다중 페이지 지문)

4. **나머지 16건**
   - Gemini 추출 품질 또는 실제 확인 필요한 경고

## Phase 2 옵션 (미착수)

### 옵션 A: segmenter 품질 개선
- 지문 헤더를 content에서 제거
- 전각공백(\u3000) → 일반 공백/줄바꿈 정규화
- 5번 선택지 이후 다음 지문 경계 처리
- 예상 효과: 텍스트불일치 ~13건 감소

### 옵션 B: Gemini 프롬프트/DPI 개선
- 지문못찾음, bracket 비결정성 감소
- PDF 페이지당 DPI 증가 (120 → 150+)
- 예상 효과: 지문못찾음 ~10건, bracket ~10건 감소
- 비용: Gemini 호출 비용 증가

### 옵션 C: 수동 검토 + DB 직접 교정
- 남은 101건 중 실제 오류와 false positive 구분
- baseline whitelist 방식으로 known issues 등록
- 예상 효과: 운영 노이즈 제거
